Why

Why should I switch from rvm to rbenv? The last few years rvm helped me a lot managing differents ruby versions and differents gemsets. Without this wonderful tool probably rails development would have been much more complicated.
So, why?
The Unix way is pretty simple. One tool should make only one small and discrete job, by this way of thought, rvm is too ambitious.Trying to manage ruby version AND the ruby gems is too much work, especially when there is another tool to do one of these works. Bundler. Bundler is absolutely the best tool ever to manage gems (is there another one?), we need only one tool to deal with different ruby version. And here we are, rbenv. Exit rvm, enter rbenv

To install rbenv you have to disintstall rvm. Don't worry about your gemset, you ain't gonna need it. Disinstall rvm {% highlight bash %} rvm implode {% endhighlight bash %} I've to remove some folder manually due to some permission problems. {% highlight bash %} Now checkout rbenv in your home direcotry. Follow the instrucion reported here https://github.com/sstephenson/rbenv#basic-github-checkout to install rbenv in your home directory, configure your bash or zsh profile, and to install ruby-build, a plugin for rbenv needed to install the wished ruby version. After that, it will be very easy install new ruby version. Type 'rbenv install -l' to have a complete list of the availables rubies. Install version 2.0.0-rc2 with {% highlight bash %} rbenv install 2.0.0-rc2 {% endhighlight bash %} It will start automatically to download the selected version. {% endhighlight bash %} Now, when you call for example'rake' rbenv pass the call to a directory in your environment PATH ~/.rbenv/shims. Shims are small executables that pass the call to the right command for that ruby version. From the rbenv website:
