--- layout: post title: "Why Processing is good for Android development" category: tags: [Processing, Android, Java, Videogames, 8bit] --- {% include JB/setup %} 2 weeks ago I have buyed this book written by Daniel Sauter, it is a book oriented to learn by doing, full of examples, that allows you to have a running application on your mobile device in less than 10 minutes. Ok,10 minues for a stupid one, neverthless, his simplicity was really surprising. But probably, this is something inside the Processing language, not only in the book.
After finishing my first very simple android application, an 8-bit looklike videogame to learn participle forms of german irregular verbs, i would like to write down the reason that have suggested me the title for this post.

Processing is based on Java

And Android too. Processing in only one abstract layer upper, but it could integrate all the functionalities provided by the Android API. The Ketai library, goes in this direction. There is still some aspects where it is better to work directly with Java (for example, parsing a JSON file), but i think that in the future we will see more processing libraries like Ketai.

Oriented to Interaction

Processing was thought to "make things talk". Touch, move, shake it, flip it, pinch it.

Low Barriers

Processing has catched the attention of lot of designers and artists. People that probably have never written an "hallo world" before know such a tool like processing. These people are now an incredible resource to build better mobile applications. Ok, people writing code without knowing how to code are like people driving a car without know how to drive a car.
